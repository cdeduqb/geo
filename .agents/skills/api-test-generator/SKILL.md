---

# 任务目标
快速为 Next.js API 路由（例如 `src/app/api/.../route.ts`）构建请求测试脚本，覆盖正常流程、参数缺失、无效参数等多种情况，方便后续集成测试或本地验证。

# 执行步骤
1. 读取用户指定的 API 路由代码，分析其接受的 HTTP 方法（GET/POST 等），以及所需的 Query 参数或 Body Payload 数据结构。
2. 在项目根目录的 `scripts/tests/` 目录下（如果不存在则创建），新建一个用于测试该接口的 `.ts` 或 `.js` 文件。
3. 使用 `fetch` 或 `axios` 编写不同场景的测试用例：
   - 场景 1：传入完美的参数，期待 `200 OK` 响应，并打印结果。
   - 场景 2：故意漏传必填字段，期待 `400 Bad Request` 和准确的错误提示。
   - 场景 3：考虑鉴权缺失或非法参数，期待 `401`、`403` 或相应的安全拦截日志。
4. 确保测试脚本可以被使用 `npx ts-node` 甚至 `node` 直接独立运行。
5. 将生成好的脚本文件路径返回给用户，并说明运行方式。
