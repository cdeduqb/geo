# 交互式迭代循环 (REPL-loop / Ralph Loop)

这个技能库引入了高效工程开发中的经典 REPL 思想。在面对环境未知的 API 调试、逻辑晦涩的函数运算、不确定的正则、或是数据结构混乱的脚本时，你不应完全依靠脑内“硬猜（Hard-guess）”，而是建立一条不断求值与验证的高效测试闭环。

## 执行指令

1. **实验性脚本化（Test-Driven Scripting）**:
   - 在处理非显态、缺乏类型或业务复杂的逻辑前，不直接修改目标文件。
   - 使用 `write_to_file` 工具创建一个独立的验证脚本（例如写到 `/tmp/test-module.js` 或 项目下的专属验证用例）。
2. **快速运行验证（Execute & Evaluate）**:
   - 编写完成最小可执行逻辑后，立即利用 `run_command`（通过 `node`、`ts-node` 或 `python`）执行它，观察返回的真实数据流形态或堆栈报错。
3. **反馈式优化（Print & Polish）**:
   - 提取执行阶段返回的真实日志或控制台 Output （这比干想有效得多）。
   - 让真实的执行结果成为你的“引导盲杖”。根据执行打印出的报错，迅速切入修改脚本，直到脚本验证完全通过为止。
4. **应用到宿主（Commit to Host）**:
   - 一旦在 REPL 循环中你的实验代码跑通并拿到了预期的干净结果（无论是一个复杂的正则、一条难以伺候的爬虫抓取器，还是数据库的抽象联查）。
   - 最后使用 `replace_file_content` 将已经得到可靠性保票的成熟逻辑完整映射并缝合到宿主文件中，最终删除试验用脚本（可清理 `/tmp`）。
