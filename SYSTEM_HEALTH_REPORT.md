## ⚠️ 警告项（需要关注）

### 1. 定时器管理 ⚠️
**问题**: 发现 42 个定时器，但只有 25 个清理  
**风险**: 可能导致内存泄漏  
**优先级**: 中  
**建议**: 
```typescript
// 在 useEffect 中确保清理
useEffect(() => {
  const timer = setInterval(() => {...}, 1000);
  return () => clearInterval(timer); // ← 必须清理
}, []);
```

### 2. 数据库连接池 ⚠️
**问题**: 未在 .env 中配置 connection_limit  
**风险**: 高并发时可能耗尽连接池  
**优先级**: 高  
**修复**: 在 `.env` 中更新：
```env
DATABASE_URL="mysql://user:pass@host:3306/db?connection_limit=10&pool_timeout=20&connect_timeout=10"
```

### 3. 硬编码密钥 ⚠️
**问题**: 发现 116 处可能的硬编码值  
**风险**: 安全隐患  
**优先级**: 中  
**建议**: 
- 将 API 密钥移到环境变量
- 使用 `process.env.XXX_KEY`
- 定期审查代码

### 4. 文件上传验证 ⚠️
**问题**: 文件上传可能缺少类型和大小验证  
**风险**: 可能被上传恶意文件  
**优先级**: 高  
**修复**: 添加验证逻辑：
```typescript
const ALLOWED_TYPES = ['image/jpeg', 'image/png', 'image/webp'];
const MAX_FILE_SIZE = 5 * 1024 * 1024; // 5MB

if (!ALLOWED_TYPES.includes(file.type)) {
  return new Response('Invalid file type', { status: 400 });
}
```

### 5. CORS 配置 ⚠️
**问题**: 未找到明确的 CORS 配置  
**风险**: 潜在的跨域安全问题  
**优先级**: 中  
**建议**: 在 `next.config.mjs` 中配置：
```javascript
async headers() {
  return [
    {
      source: '/api/:path*',
      headers: [
        { key: 'Access-Control-Allow-Origin', value: 'https://yourdomain.com' },
      ],
    },
  ];
}
```

### 6. 过时依赖 ⚠️
**问题**: 有 26 个过时的依赖包  
**风险**: 可能存在已知漏洞  
**优先级**: 中  
**修复**: 
```bash
npm outdated
npm update
npm audit fix
```

### 7. React Strict Mode ⚠️
**问题**: 未启用 React Strict Mode  
**风险**: 开发时难以发现潜在问题  
**优先级**: 低  
**修复**: 在 `next.config.mjs` 中：
```javascript
const nextConfig = {
  reactStrictMode: true,
  // ...
};
```

### 8. try-catch 不匹配 ⚠️
**问题**: try 和 catch 数量不完全匹配  
**风险**: 可能有未处理的错误  
**优先级**: 低  
**说明**: 可能是正常的（一个 catch 可以处理多个 try），建议审查异常处理逻辑
