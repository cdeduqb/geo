# 更新日志 (Changelog)

## v0.1.33 (2026-01-19)

### 优化
- **初始化脚本 (Seed) 精简**：
    - 移除了首次安装时自动创建的示例内容分类（如“科技”、“新闻”）。
    - 将默认管理员登录账号修改为：`admin`，密码修改为：`admin123`。


## v0.1.32 (2026-01-19)

### 新增与优化
- **数据库初始化优化**：
    - 在 `package.json` 中集成了 `prisma seed` 功能，支持一键初始化数据。
    - 更新了 `prisma/seed.ts`，将默认管理员账号调整为 `admin`，密码调整为 `admin23`。
- **AI 策略增强**：
    - 预设了全新的“GEO 深度内容创作策略”，专注于提升内容在生成式引擎中的权重（包括结构、事实密度、实体丰富度等维度）。


## v0.1.31 (2026-01-17)

### 新增与优化
- **爬虫数据监测**：
    - 正式开启了 AI 爬虫实时访问日志记录功能（之前处于静默模式）。
    - 优化了“活跃 AI 平台”的统计逻辑，不再仅显示前 5 名，而是展示真实的活跃平台总数。
    - 扩展了爬虫分布和热门页面的排行显示数量（Top 10）。


## v0.1.30 (2026-01-17)

### 修复与优化
- **文章一键优化增强**：
    - 重构了后端 API，引入长连接心跳机制 (Whitespace Heartbeat)，彻底解决了长文章优化时因 AI 响应过慢导致的 504 Gateway Timeout 超时问题。
    - 优化了发送给 AI 的上下文长度，在保证质量的前提下提升响应速度。
- **智能配图优化**：
    - 优化了绘图提示词生成逻辑，强制去除水印，避免生成的配图中出现类似图库或伪造的水印。
- **定时任务**：
    - 修复了 Cron 定时任务在部分环境未自动启动的问题。


## v0.1.26 (2026-01-16)

### 修复
- **多语言页眉/页脚**：修复了在文章、产品等页面未正确传递语言参数导致全局页眉显示异常的问题。
- **站点设置同步**：修复了部分系统因历史数据导致的新设置不生效问题（即使 API 路由缺失也已恢复）。
- **部署兼容性**：恢复了部署脚本中对 `schema.license.prisma` 的引用，确保与现有 License Server 部署流程兼容。


## v0.3.21 (2026-01-11)

### 关键修复
- **页面动态渲染**：启用所有页面的 `force-dynamic` 模式，确保每次请求都从数据库获取最新数据
    - 修复了首页 `page.tsx`
    - 修复了动态页面处理器 `DynamicPageHandler.tsx`
    - 修复了 `[locale]/page.tsx`

## v0.3.20 (2026-01-11)

### 修复
- 修复 `next.config.mjs` 中 `serverExternalPackages` 配置位置，消除构建警告。

## v0.3.19 (2026-01-11)

### 关键修复
- **服务器启动问题**：
    - 修复了 `output: standalone` 配置与 `npm start` 不兼容的问题
    - 现在使用 `node .next/standalone/server.js` 正确启动
    - 自动复制静态资源到 standalone 目录
- **配置警告修复**：
    - 修复 `devIndicators` 配置格式
    - 将 `serverExternalPackages` 移入 `experimental` 配置

## v0.3.18 (2026-01-11)

### 核心修复
- **服务器缓存问题**：
    - 更新脚本现在会强制清除 `.next` 和 `node_modules/.cache` 目录
    - 为所有页面添加了 `Cache-Control: no-store` 响应头，防止 Nginx/CDN 缓存
    - 添加 `Pragma: no-cache` 和 `Expires: 0` 头部作为兜底

## v0.3.17 (2026-01-11)

### 核心修复
- **模板中心**：全面重构了模板缓存刷新机制：
    - 刷新首页及所有语言版本的页面缓存
    - 自动查找并刷新所有使用该模板的页面
    - 同时刷新 Layout 和 Page 两个层级的缓存

## v0.3.16 (2026-01-11)

### 修复

    - 修复了创建产品时因 URL 唯一性约束导致的 500 错误。
    - 修复了更新产品信息（如价格、库存）后，前台页面未立即刷新的问题。

## v0.3.15 (2026-01-11)

### 修复
- **模板更新**：修复了在服务器上修改模板内容（如页眉、页脚或页面模板）后，前端页面未立即刷新的问题。

## v0.3.14 (2026-01-11)

### 核心修复
- **内容发布**：修复了在生产环境（服务器）上，创建、更新或删除页面内容后，前台页面无法立即刷新的问题（增强了 ISR 缓存刷新机制）。

## v0.3.13 (2026-01-11)

## v0.3.12 (2026-01-11)

### 修复
- **页面管理**：修复了创建重复 URL 页面导致系统崩溃的问题，增加了 URL 唯一性校验和友好的错误提示。

## v0.3.11 (2026-01-11)

### 修复
- **验证缓存清除**：禁用了站点验证文件的浏览器缓存，解决添加或修改验证文件后无法立即生效的问题。

## v0.3.10 (2026-01-11)

### 体验优化
- **即时更新**：修复了修改站点设置后首页未立即刷新的问题 (Router Cache 配置优化)。

## v0.3.9 (2026-01-11)

### 修复
- **验证参数传递**：修复了在中间件转发时导致的文件名参数丢失问题 (`File Not Found`)，增加了 Header 传参作为双重保障。

## v0.3.8 (2026-01-11)

### 重构
- **核心优化**：将站点验证机制升级为中间件动态拦截 (Middleware) 模式，不再依赖服务器文件写入权限，极大提升了在各类部署环境下的稳定性和成功率。

## v0.3.7 (2026-01-11)

### 优化
- **调试增强**：增加了站点设置保存时的文件写入日志，便于排查验证文件生成失败的问题。

## v0.3.6 (2026-01-11)

### 新增功能
- **站点所有权验证**：新增站点验证文件管理功能，支持直接导入 HTML 验证文件并物理同步到服务器，彻底解决搜索引擎验证失败的问题。
- **IndexNow 优化**：新增 IndexNow 密钥一键获取链接，指向官方生成器页面。

### 修复与优化
- **SEO 推送修复**：修复了在 IndexNow 和其他平台推送时使用了错误 URL (如 localhost) 的问题。
- **Google 推送提示**：优化了 Google Indexing API 的配置指引，明确了 Access Token 的有效期限制。
